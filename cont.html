What is Containerization?

Virtualization
It is basically an enhanced variant of Virtualization at application level. It allows multiple isolated user space istances to reun on a single kernel. These instances are referred as containers in technical terms. These Containers, provide a standard method of packaging things like, an application’s code, runtime-systems tools, system libraries and configuration into a single instance. Containers share one single kernel (essentially an OS), which is usually installed on the hardware.
Containerization is tailor made for any application that requires to be changed occasionally and be re-deployed frequently. It is best suited on Applications using Micro-services Architecture like as Alibaba provides container services for Kubernetes.
There’s another term being referred as container orchestration. It refers to organization of containers to support applications which takes help from container orchestration tools like Kubernetes, Docker Swarm and LXC.
